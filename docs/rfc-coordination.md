## How to use this file

* **RFC authors**: when status changes (draft → implemented; question resolved), update the relevant row in the same PR that lands the change. Don't let the discussion drift ahead of this file.
* **Spec readers**: this file is the entry point. The spec under [`packages/design-data-spec/spec/`](../packages/design-data-spec/spec/) is normative; RFC discussions are historical/contextual.
* **New cross-cutting questions**: open or update the relevant RFC discussion, then mirror the open question in this table.

## Phase numbering reference

The umbrella RFC ([#714](https://github.com/adobe/spectrum-design-data/discussions/714)) tracks implementation in numbered phases:

* **Phase 0** — Spec scaffolding (schemas, rule catalog skeleton, conformance harness)
* **Phase 1** — Token format, taxonomy, name object, lifecycle metadata
* **Phase 2** — Cascade resolution + dimensions
* **Phase 3** — Diff + query
* **Phase 4** — Output generators, authoring tooling (in progress)
* **Phase 5** — Distributed manifests across platform repos, upgrade automation
* **Phase 6** — Component contract: options + anatomy + state model (in progress; scoped under [RFC-A #832](https://github.com/adobe/spectrum-design-data/discussions/832))
* **Phase 7** — Foundation accessibility semantics (planned, scoped under RFC-B)
* **Phase 8** — Agent-readable surface in `sdk/` (planned, scoped under RFC-C)
* **Phase 9** — Document blocks for prose & agent context (planned, scoped under RFC-D; parallelizable with Phase 7)

Tracked on the [project board](https://github.com/orgs/adobe/projects/89).
